Tuần 1: Phân Tích Yêu Cầu và Thiết Kế Giao Diện
Ngày 1-2: Phân Tích Yêu Cầu
Member 1 (Project Lead):
Đọc và phân tích yêu cầu dự án (cả chức năng và phi chức năng).
Tạo danh sách các tính năng cần phát triển cho từng phân hệ (guest, subscriber, writer, editor, admin).
Liệt kê các API cần thiết cho từng chức năng.
Member 2 (Backend Developer):
Tạo sơ đồ cơ sở dữ liệu (ERD) cho MySQL/PostgreSQL (bao gồm các bảng: Users, Articles, Categories, Tags, Comments, Subscribers, etc.).
Xác định các quan hệ (one-to-many, many-to-many) và các ràng buộc dữ liệu.
Member 3 (Frontend/UI Designer):
Tạo wireframe cho các trang chính (Trang chủ, Danh sách bài viết, Chi tiết bài viết, Trang đăng nhập).
Chuẩn bị mockup giao diện trang chủ với carousel và slideshow.
Ngày 3-4: Thiết Kế Giao Diện
Member 1 (Project Lead):
Kiểm tra và duyệt các wireframe/mockup của Member 3.
Đảm bảo tính khả thi trong việc phát triển giao diện.
Member 2 (Backend Developer):
Thiết kế cấu trúc cơ sở dữ liệu chi tiết (bao gồm các trường và kiểu dữ liệu cụ thể cho từng bảng).
Lập kế hoạch phát triển các API cho bài viết, danh mục, người dùng, bình luận.
Member 3 (Frontend/UI Designer):
Hoàn thiện mockup trang bài viết chi tiết, trang đăng nhập và đăng ký.
Xác định các hiệu ứng động cần thiết cho giao diện (carousel, slideshow).
Ngày 5-7: Chuẩn Bị Backend và Frontend
Member 1 (Project Lead):
Xem xét lại các yêu cầu kỹ thuật và cơ sở dữ liệu.
Đảm bảo các yêu cầu xác thực người dùng (đăng nhập, đăng ký) được định hướng đúng.
Member 2 (Backend Developer):
Bắt đầu cài đặt Express.js, cấu hình cơ sở dữ liệu MySQL/PostgreSQL.
Tạo các API cơ bản cho việc hiển thị bài viết và danh mục.
Member 3 (Frontend/UI Designer):
Xây dựng giao diện tĩnh cho trang chủ, danh sách bài viết, và chi tiết bài viết.
Tích hợp hiệu ứng động (carousel, slideshow).
Tuần 2: Phát Triển Backend và Frontend
Ngày 8-10: Phát Triển Backend
Member 1 (Project Lead):
Xác định các route và yêu cầu cho API backend.
Kiểm tra việc tích hợp giữa backend và frontend.
Member 2 (Backend Developer):
Cài đặt và cấu hình Passport.js cho xác thực người dùng.
Phát triển các API cho đăng ký, đăng nhập, quản lý tài khoản người dùng.
Xây dựng API cho bài viết (lấy danh sách, tìm kiếm, chi tiết bài viết).
Member 3 (Frontend/UI Designer):
Tạo form đăng nhập và đăng ký người dùng.
Hiển thị danh sách bài viết từ API backend.
Ngày 11-13: Tích Hợp Backend và Frontend
Member 1 (Project Lead):
Quản lý việc kiểm thử và đồng bộ giữa backend và frontend.
Kiểm tra tất cả các tính năng cơ bản đã hoàn thiện.
Member 2 (Backend Developer):
Hoàn thiện API cho danh mục (category) và nhãn (tags).
Tạo API cho việc đăng bài viết và cập nhật bài viết.
Kiểm thử API và xử lý các lỗi phát sinh.
Member 3 (Frontend/UI Designer):
Hiển thị danh sách bài viết, chi tiết bài viết và các chuyên mục.
Tích hợp tìm kiếm bài viết từ API backend.
Ngày 14: Kiểm Tra và Hoàn Thiện Backend/Frontend
Member 1 (Project Lead):
Đảm bảo tất cả tính năng được triển khai và hoạt động như mong muốn.
Kiểm tra tính năng tìm kiếm và hiển thị bài viết.
Member 2 (Backend Developer):
Tinh chỉnh mã backend, kiểm thử lại tất cả các API.
Đảm bảo API bảo mật và có khả năng xử lý lỗi tốt.
Member 3 (Frontend/UI Designer):
Hoàn thiện giao diện trang chủ, thêm hiệu ứng động.
Đảm bảo giao diện hiển thị trên các thiết bị di động (responsive).
Tuần 3: Tích Hợp Các Chức Năng và Kiểm Thử
Ngày 15-17: Phát Triển Các Chức Năng Bổ Sung
Member 1 (Project Lead):
Quản lý tích hợp các chức năng (đăng bài viết, bình luận).
Xem xét và xác nhận tính năng.
Member 2 (Backend Developer):
Xây dựng API cho bình luận, phân trang bài viết.
Tạo các route cho việc quản lý tài khoản độc giả và gia hạn tài khoản.
Member 3 (Frontend/UI Designer):
Phát triển giao diện cho trang quản lý bài viết cho phóng viên, biên tập viên.
Hiển thị và kiểm thử chức năng bình luận và phân trang bài viết.
Ngày 18-20: Kiểm Thử Các Tính Năng
Member 1 (Project Lead):
Đảm bảo tất cả tính năng hoạt động đúng.
Kiểm tra tính năng bình luận và phân trang.
Member 2 (Backend Developer):
Kiểm tra và sửa lỗi backend liên quan đến bình luận và phân trang.
Đảm bảo API hoạt động ổn định và nhanh chóng.
Member 3 (Frontend/UI Designer):
Kiểm tra giao diện trang quản lý phóng viên, biên tập viên.
Tinh chỉnh giao diện và đảm bảo tính năng hiển thị đúng trên tất cả các trình duyệt.
Tuần 4: Tối Ưu Hóa, Hoàn Thiện và Deploy
Ngày 21-23: Tối Ưu Hóa và Kiểm Tra Lỗi
Member 1 (Project Lead):
Kiểm tra hiệu suất của ứng dụng và tối ưu hóa mã nguồn.
Đảm bảo mã nguồn sạch, dễ hiểu và có thể mở rộng.
Member 2 (Backend Developer):
Tối ưu hóa mã backend, đặc biệt là việc xử lý API và bảo mật.
Kiểm thử lại các API với tải trọng lớn.
Member 3 (Frontend/UI Designer):
Kiểm tra tính tương thích giao diện với các thiết bị di động và trình duyệt khác nhau.
Thêm các hiệu ứng CSS nếu cần để cải thiện trải nghiệm người dùng.
Ngày 24-26: Upload và Deploy
Member 1 (Project Lead):
Kiểm tra lại toàn bộ dự án trước khi deploy.
Đảm bảo mã nguồn đã được upload lên GitHub và quản lý đúng cách.
Member 2 (Backend Developer):
Kiểm tra và đảm bảo việc cấu hình môi trường deploy (Heroku, VPS, etc.).
Hoàn thiện API để chuẩn bị deploy lên server.
Member 3 (Frontend/UI Designer):
Tối ưu hóa giao diện cho môi trường sản xuất.
Kiểm thử giao diện trên máy chủ deploy.
Ngày 27-28: Final Testing and Go Live
Member 1 (Project Lead):
Kiểm tra toàn bộ ứng dụng trên môi trường sản xuất.
Chạy thử trên server live và xác nhận tất cả tính năng hoạt động ổn định.
Member 2 (Backend Developer):
Kiểm tra các API và dữ liệu khi chạy trên môi trường live.
Member 3 (Frontend/UI Designer):
Kiểm tra lại giao diện trên môi trường sản xuất và đảm bảo tính tương thích.
